Mine Planning Software Landscape

Task ID: PR-08 · Date: 2026-05-27 · Upstream deps: PR-01, PR-02 · Related: PR-04 (stack only — not duplicated here), PR-05, PR-07b · Themes: T1–T6

Commercial mine planning suites — GEOVIA Whittle/Surpac/MineSched, Datamine Studio OP, RPMGlobal XPAC/MinePlanner, Deswik, Micromine, Maptek Vulcan/Evolution, Hexagon MinePlan — share a common workflow from block model through pit shells, pushbacks, Gantt schedules, and plan-vs-actual reconciliation. Portfolio Angular demos should mimic planner vocabulary, horizon hand-offs, scenario comparison, and KPI dashboards in simplified form; they should not claim Whittle-grade pit optimisation, native 3D CAD, or desktop MIP engines (see PR-04 anti-patterns). This doc maps vendor roles so demos feel credible to a Ukwazi quantitative planner without reimplementing proprietary kernels.

Typical integrated workflow

Case studies and reseller workflows (e.g. Surpac → Whittle → MineSched) describe a repeatable chain. Portfolio demos can show this as a read-only pipeline diagram with synthetic hand-offs — not live CAD integration.

  1. Geology / block model — Surpac, Vulcan, Studio RM: drillholes → 3D model → block attributes (grade, density, RQD).
  2. Strategic pit optimisation — Whittle, MPPE, Vulcan pit optimiser: Lerchs-Grossmann or Pseudoflow shells; pit-by-pit NPV curves; Whittle Simultaneous Optimizer for concurrent cut-off, stockpile, blend, and schedule (industry literature: GMX/SIMO).
  3. Design & reserves — Pushback solids, bench blocks, wireframe reserves (Studio OP Reserves workflow, Vulcan Interactive Block Planner).
  4. Scheduling — Gantt + dependencies + resource leveling; scenario A/B/C (XPAC, Deswik Planning, Datamine DTS).
  5. Reporting & hand-off — Period tonnes/grade, cashflow tables, 3D animation playback, export to Excel/board packs.
  6. Reconciliation loop — End-of-period surfaces vs plan; spatial KPI dashboards (T3).

Shared data objects (synthetic-friendly)

Demos should name entities the way planners export them — aligns with PR-07b seeds.

Block model slice
Tabular: block ID, x/y/z, grade(s), material type, economic value ($/t), in-pit flag.
Pit shell / pushback
Shell ID, nested parent shell, tonnes ore/waste, strip ratio, incremental NPV.
Mining block / bench polygon
Schedulable unit with phase, bench, destination (plant / stockpile / waste dump).
Schedule period
Calendar month/quarter; planned ore/waste tonnes, grade, equipment hours, cashflow.
Scenario
Named assumption set: price, cost escalation, discount rate, capacity cap — comparable side-by-side.
Reconciliation record
Plan vs actual tonnes/grade by period or polygon; variance % and alert threshold.

Mimic vs avoid (portfolio demos)

Mimic — simplified but credible

  • Horizon navigation — tabs or stepper: Strategic → Tactical → Short-term → Reconciliation (Micromine Beyond→Alastri pattern).
  • Scenario manager — duplicate scenario, edit price/cost/capacity sliders, compare NPV/tonnes/grade in a summary table (Whittle/MPPE/XPAC all emphasise multi-scenario).
  • Pit-by-pit or phase summary chart — line/bar of incremental NPV vs shell number; static chart from precomputed JSON (not live LG; chart lib per PR-04).
  • Gantt-like period grid — rows = pushbacks or phases, columns = months; colour by material type; read-only or drag-reorder on toy instance only.
  • Constraints panel — checklist: bench precedence, MOQ, mill blend band, truck count, stockpile cap (XPAC Autoscheduler / MineSched vocabulary).
  • Assumptions audit trail — collapsible “settings report” listing every input used in a run (Whittle Pit Optimizer summary report pattern).
  • Workflow wizard — 4–6 steps with validation gates (Studio OP Reserves workflow; Datamine managed tasks).
  • Plan vs actual dashboard — KPI cards, variance heatmap by period (T3; Pitram deviation tracking analogue).
  • Destination / material routing table — source block → destination, tonnes, grade (XPAC ADS simplified).
  • Professional exports — CSV download + print-friendly summary; “board pack” PDF is optional polish.
  • Domain language in UI — pushback, bench, strip ratio, cut-off, LOM, MOQ, end-of-period surface — matches Ukwazi job ads and anchor §5.

Avoid — credibility traps

  • Claiming Whittle/Deswik integration — no logos implying certified connectors; say “inspired by industry workflow”.
  • Browser-based full pit optimisation — million-cell LG/Pseudoflow/GMX; use precomputed shells from PR-07b or <500-block toy grid (PR-04 anti-pattern).
  • Desktop-grade 3D mine CAD — full Vulcan/Surpac viewer; at most 2D bench map or lightweight Three.js pit shell with <5k faces.
  • Real-time MIP scheduling at scale — Hexagon MPSO/MPPE MILP on full block inventory; expose “optimisation run” as Cloud Function job with polling, or fixed optimal schedule.
  • Fake precision — NPV to six decimals, or optimal labels without stating toy/synthetic scope.
  • Fleet management (Pitram) scope creep — shift-by-shift dispatch GPS is execution layer; stay planning/reconciliation unless T2 DES haulage module.
  • Reimplementing vendor-specific scripting — XPAC XCM, Deswik macros; use JSON scenario configs instead.
  • Stack duplication — Firebase schema, Workers, chart libraries, route paths, and auth belong in PR-04 only; this doc names UI patterns, not implementation.

Suggested module sketch (stack in PR-04)

Frame a lazy route group as a “simplified planner workbench” — not a Whittle clone. Landing page shows the six-step industry pipeline (block model card → pit shells → phases → schedule → report → reconciliation) with each step linking to a submodule. Core views to mimic: (1) scenario manager — list/create/compare scenarios with assumption diff and pit-by-pit chart from precomputed JSON; (2) tactical schedule — period grid + constraints sidebar mirroring MineSched/XPAC language; (3) reconciliation — plan vs actual KPIs with variance alerts (T3 flagship). Any “optimise” action returns canned or server-side results — never implies live LG/MILP in-browser. Routes, Firestore schemas, Workers, chart libraries, and auth are defined in PR-04 only.
