Synthetic Data Generator Spec

Task ID: PR-07b · Date: 2026-05-27 · Upstream deps: PR-07, PR-01, PR-02 · Cross-refs: open-data-apis-mining.html, PR-05 cases, T1–T6

Three deterministic-seed generators produce mine-level CSV/GeoJSON that PR-07 public anchors cannot supply: block model (T1 scheduling topology), truck cycle (T2/T5 DES and MDP states), and plan-vs-survey variance (T3 reconciliation KPIs). Each generator accepts a JSON config, emits versioned seed files for Firebase Storage, and calibrates scale against Stats SA P2041, DMPR B1/ME Bulletin, D1 entity metadata, and USGS infrastructure priors from PR-07. T4 combines block + cycle outputs for IPCC trade-offs; T6 uses block-model entity metadata plus PR-07 closure_risk.csv — no fourth generator.

Shared conventions

  • Deterministic seeds: All generators accept integer seed (Mulberry32 or equivalent); same config + seed → identical bytes.
  • Site binding: Config references mine_id from PR-07 entities.csv (post D1 extraction) or a synthetic ID prefixed syn-.
  • Units: Metric throughout — tonnes, metres, minutes, g/t or % grade as declared in config.
  • Storage layout: seeds/{mine_id}/{generator_id}/v{semver}/ in Firebase Storage; Firestore doc syntheticSeeds/{mine_id}_{generator_id} holds config hash, anchor citations, generated-at.
  • Disclaimer: UI footer on every demo: “Synthetic data calibrated to public aggregates — not operational mine data.”
  • Post-generation validation (required): After emit, assert (1) block ore_t + waste_t ≈ tonnage_t per row; (2) sum of scheduled extract ≤ calibration cap; (3) shift actual tons in fleet_state.json within ±15% of daily_tonnage_target_t unless breakdown events explain deviation; (4) national-scale variance mean in reconciliation_daily.csv within ±0.5 pp of benchmark_kpi.csv#delta_pct over the reporting window.

Generator 1 — gen-block-model

Produces a simplified open-pit block model: bench grid, ore/waste tonnage, primary grade, geotechnical class, extraction rate limits, and bench-down precedence edges. Suitable for Gantt/NetworkX-style scheduling demos without claiming Whittle/Deswik integration. Follows standard pit-block conventions: bench-down precedence, metric units, and tonnage conservation per block (aligned with DES/block-model literature — e.g. drill-hole → block model → cycle simulation pipelines).

Generator 2 — gen-truck-cycle

Emits load–haul–dump cycle records, breakdown/repair events, and shift-level fleet state snapshots for Web Worker DES (T2) and MDP capacity demos (T5). Haul distances derive from PR-07 network skeleton or config override. Cycle phases follow discrete-event haulage practice: load → haul (loaded) → dump → return (empty), plus queue and optional breakdown (cf. OpenMines / Peru DES fleet studies). Use triangular/normal/exponential segments — not a single fixed cycle time — so queueing and fleet sizing behave realistically.

Implementation notes

  • Language: TypeScript CLI or Cloud Function generateSyntheticSeed; validate config against JSON Schema before emit.
  • Performance: Target <5 s for reference site on laptop; block model capped at 576 blocks (12×48) for browser map performance.
