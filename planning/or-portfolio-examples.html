OR Portfolio Examples

Task ID: PR-06 · Date: 2026-05-27 · Upstream deps: PR-01, PR-02, PR-04, PR-05 · Themes: T1–T6

Eighteen external operations-research portfolio demos from GitHub, Kaggle, and consultancy platforms, evaluated for UI patterns that transfer to a mining OR portfolio on Angular+Firebase (PR-04). Scope is cross-industry OR tooling and presentation — not mining case studies (see PR-05) and not South African regulatory context (PR-09). Finance “portfolio optimization” notebooks (mean–variance asset selection) are excluded; they share vocabulary but not the haulage/scheduling/reconciliation problems Ukwazi themes target.

Scope and method

  • Boundary: Public GitHub repos, Kaggle kernels, and vendor/consultancy demo UIs with OR or simulation at the core. Mining-specific citations deferred to PR-05; SA context to PR-09.
  • Portable (1–5): How readily the demo’s UI pattern ports to Angular+Firebase with Workers and synthetic data — not whether the original stack was Angular. Fourteen of 18 catalog entries score ≥4.
  • Theme tags: Which Ukwazi themes (T1–T6) the example’s interaction model best supports when re-skinned for mining KPIs.
  • Research date: May 2026 web search; live demo URLs verified where hosted (Streamlit Cloud, GitHub Pages, vendor sites).

UI patterns — what works

Patterns below appear repeatedly in high-rated examples (#01, #05–#07, #12–#15, #18) and align with dashboard UX guidance on operational coherence, progressive disclosure, and scenario comparison (ThoughtSpot, UXPin, Pencil & Paper, 2025–2026).

1. Learn / Decide route split — Tutorial route explains method; sibling Decide route runs the engine with user parameters and optional save. Proven on integraldecision.com (PR-04); maps to T2 haulage basics → fleet mix and T4 WSM method selection.

2. Scenario picker + compare — Top-bar or sidebar dropdown switches scenarios; delta KPIs vs baseline highlighted. IBM dse-do-dashboard uses DB-backed scenario sets; PathMatrix and last-mile demos compare optimized vs heuristic routes with quantified savings.

3. Sidebar parameters + explicit Run/Solve — Sliders and inputs in a fixed sidebar; user clicks Run to invoke solver (Worker or API). OpenScienceNerd SimPy template and IBM DO “Build model → Explore solution” avoid auto-rerun on every keystroke — critical for MIP/DES latency.

4. Optimizer vs baseline twin view — Side-by-side map or KPI strip showing greedy/heuristic vs OR-Tools output (#02, #03). Makes optimization value legible to non-OR reviewers — portable to haulage dispatch heuristics (PR-05 #06–#07).

5. Map + schedule dual pane — Route map (Folium, PyDeck, Leaflet) paired with Gantt or timeline (#01 PathMatrix, IBM DO scheduling Gantt). Direct analogue for truck routes + shift schedule in T1/T2 mining demos.

6. Animated entity-flow for DES — Plotly animations of entities moving through queues (#12, #13 HSMA). Surfaces bottleneck buildup faster than static histograms; strongest T2 pattern for re-skinning to load–haul–dump cycles.

7. Cards-menu demo catalog — Grid of titled cards with theme badges and deep links (integraldecision Learn index; PR-04 portable module). Scales to T1–T6 portfolio without one monolithic dashboard.

8. Inverted-pyramid KPI layout — Critical metrics top (distance, cost, utilisation), trends middle, detail tables bottom. Project Atlas and PathMatrix follow this; avoids “metric soup” on first paint.

9. Input-table review → output-table explore — IBM DO dashboard pages: Prepare Data (input DataFrames) → Explore Solution (decision variables). Builds trust with planners who expect spreadsheet-like transparency (T3 reconciliation dashboards).

10. Algorithm transparency visualizations — optimax shows branch-and-bound trees, feasible regions, and stochastic scenario profit distributions; Horizon shows Jackson network diagrams and inventory saw-tooth plots. Teaching-grade visuals double as interview credibility for T5/T6.

11. Schema-driven parameter panels — JSON/TypeScript config drives dynamic forms across demos (PR-04 ecosystem.Ai Workbench pattern). Reduces per-demo form boilerplate when shipping T1–T6 modules.

12. Exportable results — PathMatrix Excel/HTML download; batch CSV experiment upload (OpenScienceNerd template). Lets reviewers reproduce runs offline — useful for Ukwazi-style site studies without live Firebase access.

UI patterns — what fails

Notebook-only “portfolio” — Kaggle kernels (#17 gurobipy supply chain, DES Poisson tutorials) show code but no hosted interactive app. Interviewers cannot click through; fails the portfolio bar PR-04 sets for integraldecision-style demos.

Streamlit full-page rerun on every slider — Default Streamlit reruns entire script per interaction; long OR-Tools solves block UI unless isolated behind st.form + Run button or FastAPI backend (#01 PathMatrix split). PR-04 Workers + explicit Run avoid this in Angular.

Metric overload on one screen — Enterprise dashboards that cram every chart into a single view (common anti-pattern in generic BI templates) hide the one KPI the planner must act on. Split into operational vs analytical views or use tabs.

Optimizer output without baseline — Showing only the “optimal” route or schedule (#02 without comparison mode) reads as a black box. Always pair with naive policy, previous plan, or deterministic plan (PR-05 Peru DES case).

Unre-skinned domain chrome — Healthcare ED and call-centre DES repos (#12–#14) are strong technically but fail mining portfolio positioning if copy, icons, and KPIs stay clinical (PR-04 anti-pattern: gym/hospital headline).

Finance “portfolio optimization” confusion — GitHub/Kaggle repos on mean–variance asset selection (PyPortfolioOpt, MILP stock picking) dominate search results but do not demonstrate scheduling, haulage, or reconciliation — exclude from OR portfolio narrative.

LLM narrative replacing model I/O — Last-mile demo’s LLM “business report” is additive only when inputs, constraints, and KPI tables remain visible; LLM-only summaries fail technical review.

Browser-incompatible deployment — HSMA stlite apps note Firefox failures; portfolio links for interviews must declare supported browsers or use Angular+Firebase Hosting instead.

Solver on main thread — OR-Tools MIP or long DES in Streamlit’s synchronous script freezes tab; split frontend/backend (#01) or Worker (#04 stack row).

Static screenshot portfolios — README-only “dashboard” PNGs without runnable hosted URL (seen in some ML portfolio repos) do not prove OR delivery; prefer live Firebase or Streamlit Cloud links.

Example notes (selected)

01 — PathMatrix VRP T1 T2 · Portable 5

Source: rimchmielowitz/pathmatrix-app · Live: pathmatrix.streamlit.app. Streamlit UI → FastAPI → OR-Tools MIP; map routes, Gantt schedules, Excel export. Best external template for dispatch/route demo architecture on PR-04 stack (Leaflet map + Worker OR-Tools via Function optional).

02 — Last-mile logistics CVRP + agent replan T1 T2 · Portable 4

Source: Intelligent-Last-Mile-Logistics-Optimization-System. Streamlit CVRP with OR-Tools, greedy baseline vs optimized KPI strip, and natural-language replanning (capacity constraints). Pattern #4 twin-view and pattern #7 LLM-as-additive-only both validated here.

03 — NYC medical supply VRP T1 T2 · Portable 4

Source: AbhayR1104/supply-chain-optimizer. Dash + Plotly map of NYC taxi-proxy deliveries; fleet size and capacity controls; greedy vs OR-Tools CVRP comparison. Dash layout maps to PR-04 map + KPI centre pane without adopting Dash stack.

04 — Project Atlas fleet intelligence T1 T2 · Portable 3

Source: TheLogeek/Logistics-Intelligence. Streamlit + PyDeck 3D density heatmap and K-Means zone clustering for dispatch KPIs — useful for T3-style KPI pyramid (pattern #8), not a solver-first demo.

05 — optimax T1 T4 T5 T6 · Portable 5

Source: dhruvhaldar/optimax. React (Vite) + FastAPI on Vercel; visualizes LP feasible regions, B&B trees, Lagrangian convergence, two-stage stochastic farmer problem. Closest open-source analogue to a multi-method OR teaching portfolio in a modern SPA — validates React+API split PR-04 recommends for Angular.

06 — Horizon T2 T5 · Portable 5

Source: dhruvhaldar/horizon. Queueing Jackson networks (D3), inventory saw-tooth charts, TSP/VRP graphs. Strong T2/T5 visualization patterns for fleet and inventory modules without mining domain lock-in.

07 — IBM dse-do-dashboard T1–T6 · Portable 5

Source: IBM/dse-do-dashboard. Reusable Dash framework: scenario dropdown, Prepare Data / Run Model / Explore Solution / Visualization sidebar. Scenario = named input/output DataFrames in DB2. Direct pattern for Firestore scenarios/{id} snapshots in PR-04 sketch.

08 — IBM DO scheduling quick start T1 · Portable 4

Source: IBM Cloud Pak DO quick start; samples in IBMDecisionOptimization/DO-Samples. Native Gantt for scheduling experiments — layout reference for T1 pit/shift timelines; stack not portable (Cloud Pak UI).

16 — TSP routing (Streamlit + Folium tutorial) T1 · Portable 4

Source: Towards Data Science tutorial (Bruno Scalia C. F. Leite) — Pyomo MIP + OR-Tools heuristic modes, OpenStreetMap distances, streamlit-folium map. Article-first (no single canonical repo); pair with PathMatrix (#01) for full hosted reference.

17 — Kaggle / notebook supply-chain gurobipy T1 · Portable 2

Source (anti-pattern): Kaggle gurobipy kernel (viewer often unstable) · stable analogue: Gurobi supply_network_design. Listed as failure pattern #1 — code-only, no interview-clickable UI.

12–13 — HSMA SimPy DES T2 · Portable 5

Sources: Teaching_DES_Concepts_Streamlit, simpy_visualisation · Live: GitHub Pages (stlite), simpy-visualisation.streamlit.app. Animated entity flow, replication controls, resource sliders — clone interaction model for haulage DES (PR-05 #01–#03).

15 — OpenScienceNerd SimPy template T2 · Portable 5

Source: streamlit_simpy_app · Live: osnerd-simpy.streamlit.app. Sidebar params, Run button, Plotly histograms, batch CSV upload — reference implementation for DES UX checklist.

18 — integraldecision.com T2 T4 · Portable 5

Source: integraldecision.com (PR-02/PR-04 baseline). Only production example in catalog already on Angular+Firebase; cards-menu, Learn/Decide split, WSM Decide route. External Streamlit/Dash repos inform UX; this site is the stack anchor.

Consultancy platform patterns (reference only)

  • AIMMS Development (aimms.com) — integrated model + UI builder, one-step deploy; useful reference for “planner-facing” polish, not portable stack.
  • Optilogic composable apps (optilogic.com) — role-based supply-chain apps, NL app assembly; shows executive vs analyst view split relevant to T4 trade-off briefings.
  • Simfluence interactive widgets (simfluence.io) — browser what-if sliders for sales/workshop demos; pattern for T4 scenario exploration without full solver backend.
  • IBM Cloud Pak DO (quick start, DO-Samples) — native Gantt for scheduling experiments; input for T1 schedule visualization layout.
