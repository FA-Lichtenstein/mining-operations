## Key findings

- **Deliverable:** `mine-planning-software-landscape.html` — planning horizons table, vendor landscape (9 product lines), workflow chain, shared data objects, mimic-vs-avoid lists, UI pattern mimic scorecard, theme mapping, artifact scorecard, Angular sketch.
- **Horizon split is universal:** Strategic (Whittle, MPPE, Deswik GO) → tactical (MineSched, Studio OP, XPAC, Alastri ATS) → short-term (APS, Short Term Planner) → reconciliation (conformance, Pitram deviation). Demos must use this vocabulary.
- **Integrated workflow:** Surpac/block model → Whittle pit shells → pushback/reserves → Gantt schedule → board reports → plan-vs-actual loop — standard reseller narrative; demos show pipeline diagram with synthetic hand-offs only.
- **Mimic (top priority):** Scenario compare dashboard (score 5), reconciliation KPIs (5), workflow wizard (4), period Gantt grid (4), constraints panel (4), planner jargon in labels.
- **Avoid:** Browser LG/Pseudoflow/GMX, full 3D CAD, scale MIP scheduling, fake Whittle integration claims, Pitram dispatch scope, six-decimal “optimality” on toy data.
- **Vendor roles:** Whittle/MPPE = strategic NPV; MineSched/Studio OP/XPAC/Deswik = scheduling; Micromine splits Beyond (strategic) vs Alastri (tactical/short) with explicit surface hand-off; Maptek Evolution = enterprise single-dataset LOM.
- **Theme mapping:** T1 anchors on scheduling UIs; T3 on reconciliation; T4 on scenario/NPV trade-offs; T2 haulage stays separate DES modules (PR-05).
- **Data objects:** Block slice, pit shell, mining block, schedule period, scenario, reconciliation record — align PR-07b generators.
- **Artifact scorecard:** Ukwazi 5, CV 2, Stack 4, Data 4, Diff 5, Effort M.
- **PR-04 boundary respected:** Sketch mentions routes/collections only at metaphor level; no stack duplication.
