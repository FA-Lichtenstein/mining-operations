## Key findings

- **Deliverable:** `angular-firebase-or-stack.html` — executive summary, visual identity & UX, visualization library cap (D3 + ngx-echarts only), stack table, route model, engineering patterns, anti-patterns, scorecard, Angular+Firebase sketch.
- **Stack recommendation:** Angular 21 LTS standalone + Firebase (Hosting, read-only Storage, optional Functions + App Check for stretch solves); no Auth; client IndexedDB + JSON export for scenario persistence; Web Workers for LP/IP, DES, and MDP; Leaflet + GeoJSON for spatial KPIs.
- **Visual identity:** Dark charcoal base, subtle cool-accent gradients (PHNX Foundry); **two presentation modes** — workbench (four-panel, Aircraft Stability) for live sims; **notebook** (vertical cells, Observable) for narrative step-through demos; gallery on `/` tags each card with mode; Spartan/Zard + Tailwind v4.
- **Visualization (two libraries max):** **D3** for custom simulation canvases (DES animation, pit section, network flow); **ngx-echarts** for standard charts (waterfall, SPC, Gantt, heatmaps, time series). Excluded: Chart.js, ngx-charts, Plotly.
- **Routes:** `/` gallery, `/demo/:demoId` (workbench or notebook from `presentationMode` in catalog), `/about`.
- **PR-20 layout defaults:** `haulage-des`, `fleet-capacity-mdp` → workbench; `reconciliation-dashboard`, `pit-schedule-gantt` → notebook.
- **Portable engineering:** `DemoGalleryComponent`, `WorkbenchLayoutComponent`, `NotebookLayoutComponent`, schema-driven inputs, scenario JSON export/compare.
- **Anti-patterns documented:** No chart library sprawl; no large MIP in main thread; no fake Whittle/Deswik/Vulcan; no Firestore event logs; no solver keys in client; no integraldecision look-alike.
- **Scorecard (artifact):** Ukwazi 5, CV 4, Stack 5, Data 4, Diff **5**, Effort M.

## Open questions / risks

- **D3 + ECharts theming:** Shared `ukwazi-dark.json` ECharts theme and D3 colour scale must stay in sync — define CSS custom properties once.
- **Workbench responsive stack:** Four-panel grid on mobile needs explicit panel order (controls → sim → charts → metrics).
- **Lazy vs eager:** Each `/demo/:demoId` route lazy-loads; gallery shell stays lightweight for interview links.
- **Python backend scope:** Cloud Function MIP only if PR-19 shortlist explicitly selects it.
- **Map tile licensing:** Leaflet + open tiles OK for demos.
