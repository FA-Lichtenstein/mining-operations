## Key findings

- **Deliverable:** `angular-firebase-or-stack.html` — executive summary, stack table (dashboards, LP/IP, DES, MDP, maps, auth), integraldecision reuse audit, anti-patterns, scorecard, Angular+Firebase sketch.
- **Stack recommendation:** Angular 17+ standalone + Firebase (Hosting, Auth, Firestore, Storage, optional Functions); Web Workers for LP/IP, DES, and MDP; Leaflet + GeoJSON for spatial KPIs; Chart.js/ngx-charts for dashboards.
- **Compute boundaries:** Main thread for UI/charts only; Workers for OR kernels; Cloud Functions + Python OR-Tools/PuLP only when browser budget exceeded — never large MIP on main thread.
- **integraldecision routes (production bundle):** `/learn`, `/learn/stpetersburgparadox`, `/learn/weightedsummodel`, `/learn/aidisruption`, `/decide`, `/decide/weightedsummodel`, `/participate`, `/participate/replications/bottometall1989`, `/participate/replications/coxetall2009`. Components are eager-loaded in current main bundle; mining portfolio should lazy-load heavier demos.
- **Portable modules:** `cards-menu` catalog; Learn/Decide route split; per-demo feature components; Firebase bootstrap pattern; optional Participate-style run logging; ecosystem.Ai schema-driven parameter panels (JSON/formly) for cross-demo inputs.
- **Highest reuse:** WSM Learn/Decide fork for T4 method selection; cards-menu for demo registry; Worker-wrapped DES for T2 haulage re-skin.
- **Anti-patterns documented:** No large MIP in main thread; no fake Whittle/Deswik/Vulcan integration; no Firestore for full event logs; no solver keys in client; no monolithic bundle without lazy OR routes.
- **Theme mapping:** T2 DES and T5 MDP strongest browser fit; T3 maps + dashboards via GeoJSON; T1/T4 limited to toy LP/IP and WSM; T6 closure finance likely scenario spreadsheet + charts, not browser MIP.
- **Scorecard (artifact):** Ukwazi 5, CV 4, Stack 5, Data 4, Diff 4, Effort M — enables PR-19 Stack column with honest ceilings.

## Open questions / risks

- **Lazy vs eager:** integraldecision production is eager; PR-19 demos should still lazy-load to keep interview links fast on mobile.
- **Python backend scope:** Cloud Function MIP is tempting for T1 credibility — resist unless PR-19 shortlist explicitly selects it; pre-computed results may suffice.
- **Map tile licensing:** Leaflet + open tiles OK for demos; Mapbox token management if satellite basemap needed (PR-07).
- **ecosystem.Ai Workbench:** Public docs thin; schema-panel pattern inferred from CV + generic JSON-form libraries — validate against internal Workbench if access available.
- **Participate flows:** Behavioural consent/replication logging not needed for Ukwazi portfolio — omit unless differentiation requires run telemetry.
